# 中旅酒店数智化前端面试准备清单

## 1. 面试前 30 分钟

- 打开腾讯会议，检查摄像头、麦克风、网络和共享屏幕权限。
- 打开 PPT：`outputs/manual-20260513-interview/presentations/ctg-hotel-interview/output/数智化前端岗位面试作品展示.pptx`。
- 打开项目链接：`https://next-vite-delta.vercel.app/#/login`。
- 本地项目备用启动：`pnpm --filter web dev --host 127.0.0.1 --port 3088`。
- 打开 VS Code 关键文件：`home.vue`、`login.vue`、`pcDrag/index.vue`、`Modal/modalApi.ts`、`AI/index.vue`、`router/index.ts`。

## 2. 推荐展示顺序

1. 用 PPT 第 1-2 页说明：我理解这个岗位不是单纯切页面，而是服务 APP、会员体系、中台和数据驾驶舱。
2. 登录项目，展示新版登录页和首页总览。
3. 进入“低代码拖拽”，讲组件物料、画布、属性面板。
4. 进入“全局弹窗”，讲动态组件、弹窗栈、URL 状态和异常兜底。
5. 进入“AI 流式问答”，讲 SSE 流式读取、接口代理、安全输出和失败兜底。
6. 进入“权限表格/路由权限”，讲 Pinia、路由守卫、按钮级权限。

## 3. 必背回答主线

我这个项目重点证明三件事：

- 能做中后台和 H5 的基础工程：Vue3、TypeScript、Vite、Pinia、Vue Router、Element Plus。
- 能沉淀可复用能力：权限路由、自定义指令、全局弹窗、低代码拖拽组件。
- 能结合 AI 和业务：AI 流式接口、需求分析辅助、代码重构辅助、Bug 排查辅助。

## 4. 可主动提到的优化

- 已优化：登录页展示感、首页项目总览、菜单中文化、AI 输出安全和异常处理、去掉无意义健康检查请求。
- 继续优化：Lighthouse 性能指标、组件单元测试、接口统一错误码、低代码 JSON Schema、数据驾驶舱图表模块。

## 5. 被追问时的回答方向

- Hash Router：静态部署更稳，后续服务端支持 rewrite 时可切 history。
- AI 密钥安全：前端不放密钥，只请求后端代理，由服务端转发大模型。
- 低代码落地：组件配置 JSON 化，保存后端，运营可配置表单或活动页。
- 质量保障：构建检查、ESLint、核心组件测试、接口 mock、错误监控。
