# 车晓科技一面

## 1. watch 和 computed 的区别

| 方面 | computed（计算属性） | watch（监听器） |
|------|---------------------|----------------|
| 核心用途 | 根据依赖数据计算出新值（衍生数据） | 监听数据变化，执行副作用 |
| 是否有缓存 | ✅ 有缓存，依赖未变直接返回缓存值 | ❌ 无缓存，每次变化都执行 |
| 是否需要 return | 必须返回一个值 | 不需要，可执行任意逻辑 |
| 支持异步 | ❌ 不支持 | ✅ 支持（可做 API 请求） |
| 深度监听 | 自动追踪依赖，无需配置 | 需手动加 `{ deep: true }` |
| 场景示例 | 多对一：`fullName = firstName + lastName` | 一对多：搜索词变化时发请求 |

## 2. Webpack 是如何编译的

1. **初始化阶段**：读取 `webpack.config.js`，合并命令行参数，初始化 Compiler 对象
2. **编译阶段**：从 `entry` 入口解析依赖，递归分析所有依赖模块
3. **构建阶段**：建立完整模块依赖图，代码转换（ES6+ → ES5，TS → JS），处理静态资源
4. **优化阶段**：Tree Shaking 移除无用代码，代码分割，Terser 压缩混淆
5. **输出阶段**：生成 chunks，输出 bundle 文件到 `dist` 目录

## 3. Promise 的优缺点

**优点：**
- 解决回调地狱：链式调用，代码更清晰
- 更好的错误处理：统一的 `.catch()` 处理异常
- 状态不可逆：`pending → fulfilled/rejected`，避免状态混乱
- 支持并发：`Promise.all()`、`Promise.race()` 等方法
- 标准化：ES6 原生支持

**缺点：**
- 无法取消：一旦创建就会执行，无法中途取消
- 单一值：只能返回一个值
- 立即执行：创建时立即执行，不是惰性的
- 调试困难：异步栈追踪复杂
- 内存泄漏风险：长时间 pending 可能导致内存问题

---

## 4. iframe 的优缺点

**优点：**
- 隔离性：独立文档环境，样式和脚本不会相互影响
- 安全性：沙箱机制，防止恶意代码影响主页面
- 复用性：可嵌入第三方页面（地图、视频、广告等）
- 独立加载：不影响主页面的加载和渲染

**缺点：**
- SEO 不友好：搜索引擎难以索引 iframe 内容
- 性能开销：额外的 HTTP 请求和内存消耗
- 响应式困难：难以适配不同屏幕尺寸
- 通信复杂：父子页面通信需要 `postMessage`
- 用户体验差：浏览器前进后退可能失效，移动端兼容性问题

## 5. 性能优化

### 加载优化
- 代码分割：按需加载，减少首屏包体积
- 懒加载：图片、组件延迟加载
- 预加载：`<link rel="preload">` 关键资源
- CDN：静态资源加速
- 压缩：Gzip/Brotli，代码混淆

### 渲染优化
- 虚拟滚动：大列表性能优化
- 防抖节流：减少高频事件触发
- 避免重排重绘：批量 DOM 操作
- CSS 优化：避免复杂选择器，使用 `transform`

### 缓存优化
- HTTP 缓存：设置合理的 `Cache-Control`
- 浏览器缓存：`localStorage`、`sessionStorage`
- Service Worker：离线缓存
- 内存缓存：组件级缓存，避免重复计算

### 网络优化
- HTTP/2：多路复用，减少请求延迟
- 减少请求：合并文件，雪碧图
- 接口优化：数据分页，字段精简

### Vue 特定优化
- `v-show` vs `v-if`：频繁切换用 `v-show`
- `key` 优化：列表渲染使用唯一 key
- `computed` 缓存：利用计算属性缓存
- 组件懒加载：路由级别的代码分割
