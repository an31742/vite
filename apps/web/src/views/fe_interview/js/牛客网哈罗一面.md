# 牛客网哈罗一面

## 1. Vue3 封装过 Hook 吗

Hook 就是封装好的可复用方法，例如 `useDebounce`：

```js
import { ref, watch } from 'vue'

export function useDebounce(value, delay = 300) {
  const debouncedValue = ref(value.value)
  watch(value, (newValue) => {
    const timer = setTimeout(() => {
      debouncedValue.value = newValue
    }, delay)
    return () => clearTimeout(timer)
  })
  return debouncedValue
}
```

**Hook 的作用：** 让代码更模块化、可复用、易测试。

## 2. Vue2 和 Vue3 的响应式区别

| | Vue2 | Vue3 |

## 5. type 和 interface 的区别

> 本质：`type` 是"类型组合工具"，`interface` 是"结构契约"

| 维度 | interface | type |
|------|-----------|------|
| 联合类型 | ❌ 不支持 | ✅ 支持 |
| 交叉/条件类型 | ❌ | ✅ |
| 重复声明 | ✅ 自动合并 | ❌ 报错 |
| 使用场景 | API/数据结构规范 | 复杂类型运算 |

```ts
// ① 对象结构 → interface
interface User { id: number; name: string }

// ② 联合/交叉类型 → type
type Status = 'success' | 'error'
type UserWithStatus = User & { status: Status }

// ③ 基础类型别名 → 只能 type
type ID = string | number

// ④ 类实现接口 → interface 语义更强
class Admin implements User { id = 1; name = 'admin' }
```

## 6. TypeScript 泛型的作用

> 泛型 = 延迟确定类型，在使用时再决定具体类型

**三个核心要素：**
1. 复用逻辑：同一套逻辑处理不同类型
2. 类型安全：泛型 ≠ any，编译期约束
3. 类型推导：调用点决定 T

```ts
// ① 保持输入输出类型一致
function identity<T>(value: T): T { return value }

// ② 泛型约束
function getLength<T extends { length: number }>(val: T) { return val.length }

// ③ 泛型接口（后端接口建模必用）
interface ApiResponse<T> { code: number; data: T }

// ④ 默认泛型
interface Result<T = string> { data: T }

// ⑤ 条件类型（进阶）
type ReturnTypeEx<T> = T extends (...args: any) => infer R ? R : never
```

## 7. pnpm、npm、yarn 的区别

> 本质：三者都解决依赖下载 + 版本解析，但底层模型不同

- **npm**：传统"每项目一份 node_modules"
- **yarn**：在 npm 之上做性能 + 稳定性优化
- **pnpm**：全局内容寻址存储 + 硬链接，去重到极致

```
# npm/yarn
项目A/node_modules/react
项目B/node_modules/react  ← 重复存储

# pnpm
全局 store/react@18       ← 只存一份
项目A/node_modules/.pnpm  ← 硬链接
项目B/node_modules/.pnpm  ← 硬链接
```

| 维度 | npm | yarn | pnpm |
|------|-----|------|------|
| 安装速度 | 中 | 快 | 最快 |
| 磁盘占用 | 大 | 中 | 最小 |
| 幽灵依赖 | ❌ | ❌ | ✅ 避免 |
| Monorepo | 一般 | 好 | 极强 |
| 学习成本 | 最低 | 中 | 中 |
