# 综合面试题

## 1. Vue2 和 Vue3 的区别

### 响应式原理

| | Vue2 | Vue3 |
|--|------|------|
| 实现 | Object.defineProperty | Proxy |
| 对象新增属性 | 需要 `$set` | 直接赋值，自动响应 |
| 数组索引变更 | 需要 `$set` | 直接支持 |
| 性能 | 初始化递归遍历所有属性 | 懒代理，按需响应 |

### API 风格

```js
// Vue2 Options API
export default {
  data() { return { count: 0 } },
  methods: { increment() { this.count++ } },
  computed: { double() { return this.count * 2 } }
}

// Vue3 Composition API
import { ref, computed } from 'vue'
export default {
  setup() {
    const count = ref(0)
    const double = computed(() => count.value * 2)
    const increment = () => count.value++
    return { count, double, increment }
  }
}
```

### 生命周期变化

| Vue2 | Vue3 |
|------|------|
| beforeCreate / created | setup() 替代 |
| beforeMount | onBeforeMount |
| mounted | onMounted |
| beforeUpdate | onBeforeUpdate |
| updated | onUpdated |
| beforeDestroy | onBeforeUnmount |
| destroyed | onUnmounted |

### 其他重要变化

- **Fragment**：Vue3 template 支持多个根节点
- **Teleport**：将组件渲染到 DOM 任意位置
- **Suspense**：处理异步组件加载状态
- **v-model**：Vue3 支持多个 v-model，可自定义参数名
- **全局 API**：`Vue.xxx()` → `app.xxx()`
- **TypeScript**：Vue3 源码用 TS 重写，原生完整支持

### 性能优化（Vue3）

- **PatchFlags 静态标记**：diff 时跳过静态节点
- **静态节点提升（hoistStatic）**：静态节点只创建一次，复用引用
- **事件缓存（cacheHandlers）**：事件处理函数缓存，避免重复创建
- **diff 算法优化**：最长递增子序列算法，减少 DOM 移动次数

## 2. ref 与 reactive 的区别

### 核心对比

| 维度 | ref | reactive |
|------|-----|----------|
| 数据类型 | 基本类型 + 对象 | 只能是对象 |
| 访问方式 | JS 中需要 `.value`，模板自动解包 | 直接访问属性 |
| 实现原理 | 基本类型用 defineProperty，对象内部调用 reactive | Proxy |
| 替换整个值 | ✅ 直接替换，响应性保持 | ❌ 直接替换丢失响应性 |
| 解构 | ✅ 保持响应性 | ❌ 解构后失去响应性，需 toRefs |

```js
// ref 使用
const count = ref(0)
count.value++           // JS 中需要 .value
// <div>{{ count }}</div>  模板中自动解包

// reactive 使用
const state = reactive({ count: 0 })
state.count++           // 直接访问

// reactive 替换整个对象 ❌
// state = { count: 100 }  // 丢失响应性！
Object.assign(state, { count: 100 })  // ✅ 正确方式

// reactive 解构需要 toRefs
const { count } = toRefs(state)  // ✅
```

### 使用建议

- 基本类型 → 必须用 `ref`
- 需要替换整个对象 → 用 `ref`
- Vue 官方推荐：统一使用 `ref`

## 3. 大文件上传如何处理

**核心方案：分片上传 + 断点续传 + 并发控制**

### 完整流程

```
选择文件 → 计算 Hash → 询问已上传分片 → 并发上传剩余分片 → 通知服务器合并
```

### 1. 文件分片

```js
function createChunks(file, chunkSize = 5 * 1024 * 1024) { // 5MB/片
  const chunks = []
  let start = 0
  while (start < file.size) {
    chunks.push(file.slice(start, start + chunkSize))
    start += chunkSize
  }
  return chunks
}
```

### 2. 计算文件 Hash（Web Worker 避免阻塞）

```js
// 主线程
async function calcFileHash(file) {
  return new Promise((resolve) => {
    const worker = new Worker('./hashWorker.js')
    worker.postMessage({ file })
    worker.onmessage = (e) => resolve(e.data.hash)
  })
}
// hashWorker.js 中使用 spark-md5 计算 hash
```

### 3. 断点续传：询问已上传分片

```js
async function getUploadedChunks(fileHash) {
  const res = await fetch(`/api/upload/check?hash=${fileHash}`)
  const { uploadedChunks } = await res.json()
  return uploadedChunks  // 已上传的分片索引 [0, 1, 2]
}
```

### 4. 并发控制上传

```js
async function uploadChunks(chunks, fileHash, uploadedChunks) {
  const pool = []
  const maxConcurrent = 3

  for (let i = 0; i < chunks.length; i++) {
    if (uploadedChunks.includes(i)) continue  // 跳过已上传

    const task = uploadChunk(chunks[i], i, fileHash).then(() => {
      pool.splice(pool.indexOf(task), 1)
    })
    pool.push(task)

    if (pool.length >= maxConcurrent) {
      await Promise.race(pool)  // 等待最快完成的一个
    }
  }
  await Promise.all(pool)
}
```

### 5. 通知合并

```js
async function mergeChunks(fileHash, fileName, total) {
  await fetch('/api/upload/merge', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ hash: fileHash, fileName, total })
  })
}
```

## 4. 十万级数据如何处理

**核心思路：不要一次性渲染所有数据**

### 方案对比

| 方案 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| 虚拟滚动 | 长列表滚动展示 | 性能最优 | 实现复杂，定高要求 |
| 分页加载 | 表格、列表 | 简单易实现 | 用户体验一般 |
| 时间分片 | 一次性插入大量 DOM | 渐进式，不卡顿 | 数据全在内存 |
| Web Worker | 复杂数据计算 | 不阻塞主线程 | 通信有开销 |
| Object.freeze | 纯展示数据（Vue） | 节省响应式开销 | 数据不可变 |

### 方案1：虚拟滚动（最推荐）

原理：只渲染可视区域内的 DOM 节点，其余用占位撑高

```js
// 核心计算逻辑
const itemHeight = 50
const containerHeight = 500
const visibleCount = Math.ceil(containerHeight / itemHeight)
const startIndex = Math.floor(scrollTop / itemHeight)
const endIndex = startIndex + visibleCount + 1
const visibleData = data.slice(startIndex, endIndex)
const offsetY = startIndex * itemHeight  // 用 translateY 偏移
```

Vue 推荐库：`vue-virtual-scroller`、`vue3-virtual-scroll-list`

### 方案2：时间分片

```js
function timeSliceRender(data, container) {
  let index = 0
  const batchSize = 100

  function renderBatch() {
    const fragment = document.createDocumentFragment()
    const end = Math.min(index + batchSize, data.length)
    for (; index < end; index++) {
      const div = document.createElement('div')
      div.textContent = data[index]
      fragment.appendChild(div)
    }
    container.appendChild(fragment)
    if (index < data.length) requestAnimationFrame(renderBatch)
  }
  requestAnimationFrame(renderBatch)
}
```

### 方案3：Object.freeze（Vue 专用）

```js
// 纯展示数据，不需要响应式时使用
this.list = Object.freeze(largeData)  // Vue 不会为其建立响应式，节省大量内存
```

## 7. 如何封装高复用组件

**核心原则：单一职责、开放封闭、高内聚低耦合**

### 组件分类

- **基础组件**（UI 无关业务）：Button、Input、Modal
- **业务组件**（与业务强绑定）：UserCard、OrderList
- **布局组件**：Header、Sidebar、PageLayout

### Props 设计

```js
defineProps({
  size: {
    type: String,
    default: 'medium',
    validator: (v) => ['small', 'medium', 'large'].includes(v)
  },
  disabled: { type: Boolean, default: false },
  loading: { type: Boolean, default: false },
  config: { type: Object, default: () => ({}) },
})
```

### 插槽提升灵活性

```vue
<template>
  <div class="card">
    <slot name="header" />
    <slot />                                   <!-- 默认插槽 -->
    <slot name="footer" :data="innerData" />   <!-- 作用域插槽 -->
  </div>
</template>
```

### v-model 支持

```js
// 支持 v-model
defineProps({ modelValue: String })
defineEmits(['update:modelValue'])

// 支持多个 v-model
// <MyComp v-model:title="title" v-model:content="content" />
```

### expose 控制对外暴露

```js
defineExpose({ focus, reset, validate })
```

### provide/inject 跨层级通信（表单场景）

```js
// 父组件（Form）
const FormContext = Symbol('FormContext')
provide(FormContext, { disabled, size, model })

// 子组件（FormItem）
const form = inject(FormContext)
```

### Hooks 抽离逻辑

```js
// 将复杂逻辑抽成 hook，组件只负责渲染
function useTableData(fetchFn) {
  const data = ref([])
  const loading = ref(false)
  const pagination = reactive({ page: 1, pageSize: 20, total: 0 })

  async function fetchData() {
    loading.value = true
    const res = await fetchFn(pagination)
    data.value = res.list
    pagination.total = res.total
    loading.value = false
  }

  onMounted(fetchData)
  return { data, loading, pagination, fetchData }
}
```

### 高复用 Table 组件示例

```vue
<script setup>
defineProps({
  columns: Array,       // 列配置
  data: Array,          // 数据
  loading: Boolean,
  pagination: Object,   // 分页配置
  rowKey: { type: String, default: 'id' },
})
defineEmits(['page-change', 'sort-change', 'row-click'])
defineExpose({ clearSelection, toggleRowSelection })
</script>

<template>
  <div class="base-table">
    <el-table :data="data" :row-key="rowKey" v-loading="loading" @row-click="$emit('row-click', $event)">
      <template v-for="col in columns" :key="col.prop">
        <!-- 支持自定义列内容 -->
        <el-table-column v-bind="col">
          <template v-if="$slots[col.prop]" #default="scope">
            <slot :name="col.prop" v-bind="scope" />
          </template>
        </el-table-column>
      </template>
    </el-table>
    <el-pagination v-if="pagination" v-bind="pagination" @current-change="$emit('page-change', $event)" />
  </div>
</template>
```
