#  讲一下ag-grid是如何处理将12秒处理成1.2s
+ 虚拟滚动 + 懒加载 
    - 12秒场景: 一次性加载 100,000 行 → 创建 100,000 个 DOM → 渲染阻塞
    - 1.2秒场景: 只加载可见的 30 行 → 创建 30 个 DOM → 秒级渲染
+ 行高固定化  rowHeight: 30 
+ 列虚拟化
+ 禁用不必要的功能  suppressColumnVirtualisation: false  suppressRowVirtualisation: false   suppressAnimationFrame: true
+ 数据预处理 + 后端分页
+ 渲染优化  cellRenderer: (params) => params.value 
+ 批量更新数据  gridApi.updateRowData({ add: data })

# ag-grid如何处理大数据 具体原理是什么
行高固定：ag-grid 默认要求行高一致，便于计算  
懒加载：支持分页加载数据，而不是一次性加载  
列虚拟化：水平滚动时也只渲染可见列  
缓存机制：缓存已渲染的行，减少重复计算

虚拟滚动的思想：只渲染用户可见区域的行，而不是渲染所有数据。

  ]

# 讲一下nextTick的原理
核心原理：

Vue 使用微任务队列实现 nextTick

数据修改 → 加入更新队列 → 调用 nextTick

同步代码执行完 → 执行 nextTick 回调 → DOM 已更新

关键机制：

批量更新：多个数据修改只触发一次 DOM 更新

去重：同一个 watcher 只在队列中出现一次

排序：按创建顺序执行，父组件先于子组件

+ 核心概念事件循环
+ ┌─────────────────────────────────────────┐  
│         JavaScript 事件循环              │

├─────────────────────────────────────────┤  
│ 1. 执行同步代码（Call Stack）           │  
│ 2. 执行微任务队列（Microtask Queue）    │  
│    - Promise.then()                     │  
│    - nextTick()                         │  
│    - MutationObserver                   │  
│ 3. 执行宏任务队列（Macrotask Queue）    │  
│    - setTimeout                         │  
│    - setInterval                        │  
│    - requestAnimationFrame              │  
└─────────────────────────────────────────┘

+ Vue 的更新流程  
// 步骤 1: 修改数据  
this.count++

// 步骤 2: 触发 setter（响应式系统）  
// 在 setter 中调用 dep.notify()

// 步骤 3: 通知所有 watcher  
// watcher.update() 被调用

// 步骤 4: 将 watcher 加入更新队列  
// queueWatcher(watcher)

// 步骤 5: 调用 nextTick(flushSchedulerQueue)  
// 将批量更新函数加入 nextTick 队列

// 步骤 6: 同步代码执行完毕

// 步骤 7: 执行 nextTick 回调  
// flushSchedulerQueue() 执行  
// 触发 watcher.run()  
// 更新 DOM

// 步骤 8: 执行用户的 nextTick 回调  
// 此时 DOM 已更新

+ 
    1. 源码级实现（Vue3）

```plain
// 简化版 Vue3 nextTick 源码
let callbacks = []
let timerFunc
let pending = false

export function nextTick(fn) {
  return new Promise((resolve) => {
    // 将回调加入队列
    callbacks.push(() => {
      if (fn) {
        try {
          fn()
        } catch (e) {
          handleError(e)
        }
      }
      resolve()
    })
    
    // 如果还没有调度，则调度
    if (!pending) {
      pending = true
      timerFunc()
    }
  })
}

// 选择最优的微任务实现
if (typeof Promise !== 'undefined') {
  // 优先使用 Promise（最快的微任务）
  timerFunc = () => {
    Promise.resolve().then(flushCallbacks)
  }
} else if (typeof MutationObserver !== 'undefined') {
  // 降级方案：MutationObserver
  timerFunc = () => {
    const observer = new MutationObserver(flushCallbacks)
    const textNode = document.createTextNode('0')
    observer.observe(textNode, { characterData: true })
    textNode.data = '1'
  }
} else {
  // 最后降级：setTimeout
  timerFunc = () => {
    setTimeout(flushCallbacks, 0)
  }
}

function flushCallbacks() {
  pending = false
  const copies = callbacks.slice(0)
  callbacks.length = 0
  
  for (let i = 0; i < copies.length; i++) {
    copies[i]()
  }
}
```

// Q: 为什么需要 nextTick？  
// A: Vue 的更新是异步的，nextTick 确保 DOM 已更新

// Q: nextTick 和 setTimeout 的区别？  
// A: nextTick 是微任务，setTimeout 是宏任务，nextTick 更快

// Q: 多个 nextTick 会执行多少次？  
// A: 只执行一次 timerFunc，但所有回调都会执行

// Q: nextTick 中能获取到最新的 DOM 吗？  
// A: 能，因为此时 DOM 已更新

// Q: 能在 nextTick 中修改数据吗？  
// A: 能，但会触发新的更新，需要再次 nextTick



# 讲一下vue2编译原理
模板字符串  
    ↓  
┌─────────────────────────────────────┐  
│ 1. 解析（Parse）                    │  
│    将模板转换为 AST（抽象语法树）   │  
└─────────────────────────────────────┘  
    ↓  
┌─────────────────────────────────────┐  
│ 2. 优化（Optimize）                 │  
│    标记静态节点和静态根节点         │  
└─────────────────────────────────────┘  
    ↓  
┌─────────────────────────────────────┐  
│ 3. 生成（Generate）                 │  
│    将 AST 转换为渲染函数代码        │  
└─────────────────────────────────────┘  
    ↓  
渲染函数

完整编译流程示例

```plain
// 原始模板
const template = `
  <div id="app">
    <h1>{{ title }}</h1>
    <ul>
      <li v-for="item in items" :key="item.id">
        {{ item.name }}
      </li>
    </ul>
  </div>
`

// 步骤1：解析
const ast = parse(template)
// 得到 AST 树

// 步骤2：优化
optimize(ast)
// 标记静态节点

// 步骤3：生成
const code = generate(ast)
// 得到渲染函数代码

// 最终生成的渲染函数
function render() {
  with(this) {
    return _c('div', { attrs: { id: 'app' } }, [
      _c('h1', [_v(_s(title))]),
      _c('ul', _l((items), function(item) {
        return _c('li', { key: item.id }, [_v(_s(item.name))])
      }), 0)
    ])
  }
}


```

```plain
// 编译时（构建时）
// - 模板 → AST → 渲染函数
// - 优化静态节点
// - 生成最优代码

// 运行时（浏览器中）
// - 执行渲染函数
// - 生成虚拟 DOM
// - 进行 diff 和更新

// 优势：
// - 编译时做优化，运行时更快
// - 减少运行时的计算
```

三个阶段：

解析（Parse）：模板 → AST

优化（Optimize）：标记静态节点

生成（Generate）：AST → 渲染函数

关键概念：

AST：抽象语法树，表示模板结构

静态节点：不会变化的节点，可以优化

渲染函数：最终执行的函数，生成虚拟 DOM

性能优化：

静态提升：复用静态节点

预字符串化：合并连续的静态节点

编译缓存：避免重复编译

编译输出：

_c：createElement

_v：createTextVNode

_s：toString

_l：renderList

_e：createEmptyVNode

# vue2中data为什么是一个函数而不是一个对象
核心答案：  
避免数据污染：如果 data 是对象，多个组件实例会共享同一个对象  
每个实例独立：data 是函数，每次调用都返回新对象，确保每个实例有独立的数据  
Vue 的设计：Vue 在创建实例时会调用 data 函数获取数据

关键点：  
data 必须是函数，不能是对象  
每个组件实例都有独立的 data 副本  
这是 Vue 的强制规范

对比：  
data：必须是函数（返回对象）  
methods：是对象（值是函数）  
computed：是对象（值是函数）

原因：  
状态（data）需要隔离  
行为（methods、computed）可以共享



# vue生命周期请求在哪里为什么
场景	钩子	原因  
只需要数据	created	更早获取，减少白屏  
需要操作 DOM	mounted	DOM 已挂载  
响应数据变化	watch	监听数据变化  
定时请求	mounted	初始化定时器  
核心原因：

created：数据已初始化，可以赋值，但 DOM 未挂载  
mounted：DOM 已挂载，可以操作 DOM，获取 DOM 信息  
updated：容易无限循环，不适合发起请求  
beforeCreate/beforeMount：数据或 DOM 未准备好

最常用：  
mounted 是最安全、最常用的选择  
created 用于只需要数据的场景  
watch 用于响应数据变化

# keepalive的实现原理是什么
KeepAlive 是 Vue 内置组件，通过缓存组件实例而不是销毁重建来保持状态

```
// 简化版原理
class KeepAlive {
  constructor() {
    this.cache = new Map();      // 缓存组件实例
    this.keys = new Set();       // 记录缓存 key
  }

  render(vnode) {
    const key = vnode.key || vnode.type;
    
    if (this.cache.has(key)) {
      // ✅ 命中缓存，复用实例（不会重新 mounted）
      vnode.component = this.cache.get(key);
      vnode.shapeFlag |= ShapeFlags.COMPONENT_KEPT_ALIVE;
    } else {
      // ❌ 未命中，创建新实例并缓存
      this.cache.set(key, vnode.component);
      this.keys.add(key);
    }
    
    return vnode;
  }

  // LRU 缓存淘汰（超过 max 时删除最久未使用的）
  prune() {
    if (this.keys.size > this.max) {
      const firstKey = this.keys.values().next().value;
      this.cache.delete(firstKey);
      this.keys.delete(firstKey);
    }
  }
}
```
