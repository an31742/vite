# 联想一面

## 1. Vue 编译过程

**构建时编译：**
1. 将模板字符串编译成 AST 抽象语法树
2. 遍历 AST，标记静态节点
3. 根据优化后的 AST 生成 render 函数

**运行时编译：**
1. 使用 Proxy 创建响应式数据
2. 执行 render 函数生成虚拟 DOM
3. 根据 diff 算法结果生成 patch 更新真实 DOM

## 2. Vue 中 key 的作用

key 是虚拟 DOM 节点的唯一标识，让 diff 算法更准确地比较新旧虚拟 DOM 树。

## 4. Webpack HMR vs Vite HMR

### Webpack HMR 流程

文件变化 → webpack 重新编译 → 生成 `hot-update.js` → WebSocket 推送 → 浏览器接收更新

```js
// Webpack HMR 配置
module.exports = {
  devServer: { hot: true },
  plugins: [new webpack.HotModuleReplacementPlugin()]
};

// HMR API
if (module.hot) {
  module.hot.accept('./component.js', () => {
    console.log('组件已更新');
  });
}
```

### Vite HMR 流程

文件变化 → esbuild 快速转换 → WebSocket 推送 → 浏览器原生 import 更新

```js
// Vite HMR API
if (import.meta.hot) {
  import.meta.hot.accept((newModule) => {
    console.log('模块已更新');
  });
}
```

### 对比

| 特性 | Webpack HMR | Vite HMR |
|------|-------------|----------|
| 编译方式 | 打包整个应用 | 按需编译单个文件 |
| 更新速度 | 较慢 | 极快（esbuild + ESM）|
| 模块系统 | CommonJS/AMD | 原生 ESM |
| 状态保持 | 需要手动处理 | 自动保持组件状态 |
| 配置复杂度 | 需要配置插件 | 开箱即用 |

## 8. Vue nextTick

nextTick 是 Vue 提供的 API，用于在下次 DOM 更新循环结束之后执行延迟回调。

**优先级：** Promise > MutationObserver > setImmediate > setTimeout

```js
// 简化实现
const callbacks = [];
let pending = false;

function nextTick(cb) {
  callbacks.push(cb);
  if (!pending) {
    pending = true;
    Promise.resolve().then(flushCallbacks); // 微任务
  }
}
```

## 14. 虚拟 DOM 原理

虚拟 DOM 就是 JavaScript 对象，Vue 通过它实现高效的声明式渲染和差异化更新。

1. **生成 VNode**：通过 render 函数（h 函数）返回 VNode 树
2. **首次渲染（Mount）**：遍历 VNode 树创建真实 DOM
3. **更新（Update）**：数据变化 → 重新执行 render 生成新 VNode → Diff 比较 → Patch 只更新变化部分
