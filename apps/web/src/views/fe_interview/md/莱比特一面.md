### 1. 服务端渲染 (SSR) 和普通开发 (CSR) 有啥区别

#### 本质
**本质是“首屏 HTML 的生成时机和场所不同”**。

- CSR：浏览器拿到空 HTML + JS，在客户端执行 JS 拼装 DOM，然后渲染。
- SSR：服务器请求数据后，直接拼好完整的 HTML 返回给浏览器，浏览器只负责渲染和后续交互的“激活”（hydrate）。

#### 核心区别

| 维度 | CSR | SSR |
|------|-----|-----|
| **首屏速度** | 慢（需等 JS 下载、解析、执行） | 快（直接返回有内容的 HTML） |
| **SEO** | 差（爬虫可能看不到内容） | 好（直接抓取完整 DOM） |
| **服务器压力** | 小（静态文件托管即可） | 大（需服务器动态拼 HTML） |
| **交互延迟** | 首次加载后交互快（SPA） | 首屏可见但交互需等待 JS hydrate，可能“假活” |
| **开发复杂度** | 简单，纯前端 | 复杂，需处理 Node 服务端环境、同构代码、状态同步 |

#### 为什么要有 SSR（结合你的项目）
你做过 SSR，所以必须讲出真实痛点：
“我们做 ToB 的 AI 对话系统，本身不需要 SEO，但我们需要**首屏极快**。如果用 CSR，用户打开对话页面时先白屏 1-2 秒等 JS 加载，再请求历史消息，体验很差。用 SSR 后，首屏直接渲染历史消息列表，用户瞬间看到内容，然后 Vue 做 hydrate 接管交互。我们还特意做了流式 SSR，渲染关键骨架后立刻下发给浏览器，TTFB 降低 50%。”

#### 面试陷阱
- 不要说“SSR 就是服务端渲染”，要讲出**同构**和 **hydrate**。
- 不要忽略 SSR 的代价：服务器成本、状态窗口不一致（如 `window` 不可用）、第三方库兼容。
- 高级表达： “SSR 本质是用服务器 CPU 换用户感知速度，用架构复杂度换 SEO 和首屏体验。”

### 2. jQuery 的 `.on('click')` 和 `.click()` 有啥区别

#### 本质
- `.click()` 是简写，直接给当前已存在的元素绑定 click 事件。
- `.on('click')` 是完整的**事件委托**入口，不仅能绑定当前元素，还能给**动态添加的元素**绑定事件。

#### 核心区别
1. **动态元素处理**（这是面试核心）：
   ```js
   // 以后动态添加的 .item 不会有事件
   $('.item').click(function(){});
   
   // 动态添加的 .item 也能触发
   $(document).on('click', '.item', function(){});
   ```
   原理：`.on()` 利用事件冒泡，委托在祖先元素上，通过 `event.target` 匹配选择器。

2. **性能**：如果页面上有 1000 个按钮，`.click()` 会绑定 1000 个事件处理函数；`.on()` 只绑定一个，内存更省。

3. **源码角度**：`.click()` 内部其实就是调用的 `.on('click', ...)`，但缺少了 `selector` 参数，所以没有委托能力。

#### 高级回答
“面试官，这个问题看似简单，其实关系到 DOM 事件模型和内存管理。Vue 的事件绑定 `@click` 在编译后，底层也是调用 `addEventListener`，并且自动做了事件委托优化和组件销毁时的自动解绑。jQuery 的 `.on()` 通过委托机制解决了动态元素的监听，这是早期前端框架渲染列表时常用手段，现在虚拟 DOM 框架内部已帮我们处理了。”

### 3. 写一个 CSS 运动从做到右 10 秒动画效果

```css
.box {
  width: 100px;
  height: 100px;
  background: #409eff;
  animation: moveRight 10s linear infinite alternate;
}

@keyframes moveRight {
  0% {
    transform: translateX(0);
  }
  100% {
    transform: translateX(calc(100vw - 100px)); /* 移到最右 */
  }
}
```

#### 追问：为什么用 `transform` 而不是 `left`？
**本质**：`transform` 只触发合成（Composite），不触发布局（Layout）和绘制（Paint）。`left` 会触发 Layout，导致每一帧都重排，性能极差。这就是浏览器的渲染流水线原理。

### 4. 写一个倒着的三角形

```css
.triangle-down {
  width: 0;
  height: 0;
  border-left: 50px solid transparent;
  border-right: 50px solid transparent;
  border-top: 50px solid #409eff; /* 上边框颜色和宽度决定三角形状 */
}
```

如果是正三角形朝上，用 `border-bottom` 代替 `border-top`。原理是利用边框交汇处斜线来模拟三角形。
