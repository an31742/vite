### 1. SSE 和 WebSocket 有啥区别

**本质**：SSE 是“单向流式文本推送”，WebSocket 是“全双工二进制通信”。

**核心区别**：
- **通信方向**：SSE 仅服务端 → 客户端；WebSocket 双向。
- **底层协议**：SSE 是标准 HTTP，浏览器原生 `EventSource` 支持；WebSocket 是独立协议，需 HTTP 握手后升级。
- **数据格式**：SSE 纯文本 `text/event-stream`；WebSocket 支持文本和二进制帧。
- **复杂度**：SSE 实现极简，天然支持自动重连；WebSocket 需手动处理心跳、重连、帧解析。
- **适用场景**：SSE 适合 AI 流式输出、股票推送、进度条；WebSocket 适合聊天、协作编辑、实时游戏。

**我的实践**：“我们 AI 对话用 SSE 而不是 WebSocket，因为场景是服务端单向推生成内容，SSE 足够且简单。我还用 Fetch + ReadableStream 替代原生 `EventSource`，实现了主动中断生成的功能。”

### 2. SSE 的原理是什么

**本质**：SSE 是“基于 HTTP 长连接的流式文本协议，服务器保持响应不结束，持续向客户端写数据”。

**核心流程**：
1. 客户端发起 HTTP GET 请求，请求头 `Accept: text/event-stream`。
2. 服务端返回响应头 `Content-Type: text/event-stream`，并保持连接不关闭。
3. 服务端持续向连接写入数据，格式为 `data: xxx\n\n`，双换行是分隔符。
4. 浏览器 `EventSource` 监听 `onmessage` 事件，每次收到 `data:` 行就触发。
5. 连接断开时浏览器自动重连（`Last-Event-ID` 头实现断点续传）。

**我的实践**：“我们没用 `EventSource`，因为需要 POST 请求和主动中断。改用 `fetch` 获取 `response.body` 的 ReadableStream，在 `while(true)` 循环里 `reader.read()`，用 `TextDecoder` 解析 SSE 格式，遇到 `[DONE]` 标识结束。”

### 3. Vue2 和 Vue3 有啥区别

**本质**：响应式机制从“对象属性劫持”升级为“对象代理”，编程范式从“选项分割”升级为“逻辑聚合”。

**核心三点**：
- **响应式**：Vue2 `Object.defineProperty` 无法检测属性增删、数组索引修改需重写方法；Vue3 `Proxy` 完整代理，支持 Map/Set，惰性劫持性能更好。
- **API 组织**：Vue2 Options API 按 data/methods/computed 分割，逻辑分散；Vue3 Composition API 按功能聚合，`setup()` 里一把梭，组合函数天然解耦逻辑复用。
- **编译优化**：Vue3 静态提升、PatchFlags 靶向更新、Fragment 多根节点，diff 更快。

**我的感受**：“迁移 Vue3 后最爽的是代码复用。以前 mixins 有命名冲突，现在 Composables 一个函数搞定权限判断、流式输出解析，到处引用，清晰且可测试。”

### 4. 前端的权限管理是如何实现的

**本质**：RBAC（基于角色的访问控制），通过服务端下发的权限标识，动态过滤前端路由表和界面元素。

**核心实现**：
- **路由权限**：后端返回权限数组，前端在 `router.beforeEach` 里动态 `addRoute` 用户有权访问的路由。
- **菜单权限**：递归过滤路由配置，生成侧边栏菜单。
- **按钮权限**：封装 `v-permission` 指令，绑定权限标识，没有则 `removeChild`。
- **数据权限**：后端过滤返回，前端只做展示控制。

**关键点**：“权限数据刷新后要从缓存恢复。真实权限校验在后端，前端只做 UI 层的显示/隐藏，不能依赖前端拦截保安全。”

### 5. 低代码平台如何开发

**本质**：基于“Schema 驱动的渲染引擎”，将 JSON 描述的页面结构动态解析为真实组件树。

**核心架构**：
- **Schema 协议**：每个节点定义 `component`（组件名）、`props`（属性）、`children`（子节点）、`events`（事件绑定）。
- **渲染器**：核心组件递归遍历 Schema，用 `<component :is>` 动态渲染。
- **物料中心**：维护组件名到真实组件的映射表，支持异步加载远程物料。
- **编辑器**：可视化拖拽生成 Schema，实时预览。

**我的实践**：“我们低代码平台用 JSON Schema 描述页面，渲染器通过 `provide/inject` 注入上下文方法，Schema 里配置事件名字符串，渲染时从上下文取真实函数绑定。这样完全解耦了组件行为和业务逻辑。”

### 6. AG-Grid 如何进行性能优化

**本质**：核心是把“巨量数据的 Vue 响应式劫持”砍掉，并启用 AG-Grid 自身的虚拟化能力。

**三大关键**：
- **切断响应式**：用 `shallowRef` 或 `markRaw` 包裹 `rowData`，避免 Vue 深度代理万级数据。这就是我们12秒降到1.2秒的关键。
- **虚拟滚动**：启用 `rowModelType: 'infinite'`，只渲染视口内行，DOM 数量从万级降到百级。
- **列定义优化**：`Object.freeze` 列定义对象，`cellRenderer` 用函数式组件或纯函数，减少组件实例化。

**效果**：“经过这三步，AG-Grid 十万条数据首帧渲染从卡死变为1秒内，滚动帧率稳定 60fps。”
