### 1. 低代码开发维护的 JSON 如何在页面渲染成组件

#### 本质
本质是 **“Schema 驱动的递归渲染引擎”**。JSON 描述界面结构，渲染器遍历 JSON 节点，动态解析为 Vue 组件树。

#### 核心设计
1. **Schema 协议**：每个 JSON 节点必含 `component`（组件名）、`props`、`children`（嵌套数组）。
2. **渲染器**：
   - 一个核心组件 `SchemaRenderer`，接收 `schema` 对象。
   - 使用 `<component :is="schema.component" v-bind="schema.props" v-on="resolveEvents(schema.on)">`。
   - 递归：如果 `schema.children` 存在，在内部继续使用 `SchemaRenderer` 渲染。
3. **物料映射**：维护一份 `componentMap`，将字符串名映射到真实组件。例如 `{ 'ag-grid': AgGrid, 'el-input': ElInput }`。

#### 项目实战
“我们低代码平台的渲染器，我通过 `component :is` 实现无限递归。遇到的问题是：**事件传递**。JSON 里只能描述事件名字符串，如 `{ "on": { "click": "handleClick" } }`，需要解析成实际函数。做法是：在父容器提供 `eventBus` 或通过 `provide` 注入方法集，渲染器根据 `on` 配置动态生成函数并绑定到组件上。这样，可视化搭建的页面，最终完全是实时渲染出来的，零硬编码。”

### 2. component 动态渲染组件

#### 本质
`<component :is="xxx">` 的本质是 Vue 内置的 **“动态组件挂载点”**，内部通过 `resolveDynamicComponent` 将 `is` 的值（字符串组件名/组件对象/异步组件）动态解析成 VNode 的 type。

#### 源码级运行
- 当 `is` 变化时，触发 `patch`，如果新旧类型不同，会卸载旧组件，挂载新组件。
- 常与 `<keep-alive>` 配合，缓存组件实例避免重复销毁/重建。

#### 应用
我们利用 `<component>` 做 **Tab 切换、低代码渲染、多态组件**。例如一个根据 `field.type` 动态渲染不同表单控件的表单生成器，代码极简。

### 3. 动态组件 component 如何动态传递渲染事件

#### 本质
动态传递事件就是 **动态绑定 `$listeners` 或计算事件对象**。

#### 方法
1. **v-on 对象语法**：Vue3 中可使用 `v-on="eventObject"`，其中 `eventObject` 是一个对象，键为事件名，值为处理函数。
   ```html
   <component :is="comp" v-bind="props" v-on="dynamicEvents" />
   ```
2. **事件解析**：根据 Schema 里配置的事件字符串，从当前上下文（通过 `provide/inject` 或 props）中获取真实函数，组成对象传递给 `v-on`。
3. **Vue2**：使用 `v-on="$listeners"` 做完整事件的透传，这是封装高阶组件的基础。

#### 项目落地
低代码平台事件配置如：`{ "on": { "rowClick": "onRowClick" } }`，渲染器通过 `inject` 取得上下文方法，拼成 `{ rowClick: context.onRowClick }` 赋给 `v-on`。

### 4. Vue 常用指令

快速过，但要讲出几个有深度的：
- **v-bind / v-model**：单向绑定/双向绑定，v-model 是语法糖。
- **v-if / v-show**：编译时移除 vs 仅切换 display。
- **v-for**：必须用 key，配合 Diff 算法。
- **v-on**：事件绑定，支持修饰符。
- **v-html / v-text**：直接插入 HTML（XSS 风险）。
- **v-once / v-memo**：性能优化用。
- **自定义指令**：下面重点拆解。

### 5. 插槽如何传递值（作用域插槽核心）

#### 本质
作用域插槽本质是“**父组件决定子组件内容的渲染，但子组件将内部数据暴露给父组件**”，实现数据反向传递。

#### 使用
子组件：
```html
<slot :item="item" :index="index" />
```
父组件：
```html
<child v-slot="slotProps">
  {{ slotProps.item.name }}
</child>
```
编译后，子组件的 `slot` 返回一个 VNode 生成函数，父组件传递的函数接受参数生成内容。

#### 项目
AG-Grid 的单元格渲染完全依靠作用域插槽，父组件拿到行数据 `params` 后自定义渲染。

### 6. 自定义指令（深度）

#### 本质
自定义指令本质是 **“直接操作 DOM 的钩子函数”**，复用非响应式的底层 DOM 行为。

#### 注册
- **全局**：`app.directive('focus', { ... })`
- **局部**：组件里 `directives: { focus: { ... } }`

#### 生命周期（Vue3）
- `created`：绑定元素属性初始化前。
- `mounted`：元素插入 DOM。
- `updated`：组件 VNode 更新后（可能子节点未更新）。
- `beforeUnmount` / `unmounted`：卸载前/后。

#### 封装实践
我封装了 `v-permission` 做按钮权限控制：
```js
app.directive('permission', {
  mounted(el, binding) {
    const { value } = binding;
    if (!hasPermission(value)) {
      el.parentNode?.removeChild(el);
    }
  }
});
```
还封了 `v-loading`、`v-copy` 等，核心是复用 “操作真实 DOM” 的逻辑，无需每处都写 ref 操作。

#### 源码理解
指令会被编译成 `withDirectives` 包装的 VNode，在 `patch` 过程中调用对应钩子。

### 7. 路由模式

#### 本质
路由模式的本质是“**如何通过 URL 变化驱动前端视图切换，并处理浏览器历史记录**”。

#### 三种模式
1. **Hash 模式**  
   - URL 带 `#`，`#` 后内容不发送服务器。  
   - 实现：监听 `hashchange` 事件，解析 `location.hash`。  
   - 优：不需要服务器配置，兼容性好。  
   - 劣：SEO 不友好，URL 丑。

2. **History 模式**  
   - 基于 HTML5 History API（`pushState`, `replaceState`, `popstate`）。  
   - URL 干净，无 `#`。  
   - 必须服务器配置：所有路径 fallback 到 `index.html`，否则刷新 404。  
   - 适合注重 SEO 或 URL 美的项目。

3. **Memory 模式（抽象模式）**  
   - 用于非浏览器环境（如 SSR、Node 测试），路由状态存在内存中。

#### 项目
我们微前端主应用用 History 模式，子应用通过主应用注入的 base 来独立路由空间，完美共存。
