# 联想消费者团队二面

## 1. 如何识别需不需要做成一个组件

**判断标准：**
- 是否会在多处使用？（复用性）
- 是否有独立的 props/emit 接口？（独立性）
- 是否有独立的状态管理？（可维护性）
- 是否需要单独优化性能？（可测试性）
- 是否代表明确的业务概念？（业务语义）

---

## 2. AI 对话框可以抽象的模块

**UI 层：**
- 消息气泡组件（文本、代码、图片、表格等多种格式）
- 输入区域组件（多行输入、快捷指令、文件上传）
- 加载状态组件（打字机效果、思考中动画）
- 工具栏组件（复制、重新生成、点赞/点踩）

**功能层：**
- 消息管理器（消息队列、历史记录、滚动定位）
- 会话管理器（多会话切换、上下文管理）
- 响应处理器（流式响应、错误处理、重试机制）

**业务层：**
- API 适配器（不同 AI 服务接口统一）
- 权限控制器（敏感词过滤、调用频率限制）

## 3. 修改组件如何保证不影响已集成的页面

1. **版本控制**：语义化版本（SemVer），破坏性变更升级大版本号
2. **向后兼容**：新增功能用新 props，保留旧 props，废弃功能标记 `@deprecated`
3. **渐进式迁移**：提供迁移脚本，发布过渡版本同时支持新旧 API
4. **测试保障**：单元测试、集成测试、快照测试检测 UI 变化
5. **文档沟通**：更新 CHANGELOG，提供 Migration Guide，提前通知相关方
6. **特性开关**：新功能通过 flag 控制灰度，支持回滚

## 4. 完整的项目开发流程

1. **需求阶段**：PRD 评审、技术方案设计、工作量评估排期
2. **设计阶段**：UI/UX 设计稿评审、交互原型确认、API 设计
3. **开发阶段**：Git 分支管理（feature/xxx）、编码实现、自测、Code Review

## 6. H5 调用原生能力

### JSBridge 方案（推荐）

```js
class DeviceAdapter {
  detectPlatform() {
    const ua = navigator.userAgent;
    if (/iPhone|iPad|iPod/i.test(ua)) return 'ios';
    if (/Android/i.test(ua)) return 'android';
    return 'web';
  }

  call(method, params) {
    const platform = this.detectPlatform();
    if (platform === 'ios') {
      window.webkit?.messageHandlers?.jsBridge?.postMessage({ method, params });
    } else if (platform === 'android') {
      window.AndroidJSBridge?.call(method, params);
    }
  }
}
```

### 其他方案
- **URL Scheme**：`window.location.href = 'myapp://camera/scan'`
- **Universal Links / App Links**：现代方案，配置关联文件

## 7. 设计模式

```js
// 单例模式
class Singleton {
  static instance = null;
  constructor() {
    if (Singleton.instance) return Singleton.instance;
    Singleton.instance = this;
  }
}

// 观察者模式（EventBus）
class EventBus {
  constructor() { this.events = {}; }
  on(event, fn) { (this.events[event] ||= []).push(fn); }
  emit(event, ...args) { (this.events[event] || []).forEach(fn => fn(...args)); }
}

// 适配器模式
class PaymentAdapter {
  constructor(oldPayment) { this.oldPayment = oldPayment; }
  pay(type, amount) {
    if (type === 'alipay') return this.oldPayment.payWithAlipay(amount);
  }
}

// 策略模式
const strategies = { A: (b) => b * 4, B: (b) => b * 2, C: (b) => b * 1 };
function calculateBonus(level, bonus) { return strategies[level](bonus); }

// 代理模式（Vue3 响应式原理）
const proxy = new Proxy(target, {
  get(target, key) { /* 依赖收集 */ },
  set(target, key, value) { /* 触发更新 */ }
});
```

## 9. 线上紧急 Bug 处理流程

1. **快速响应（5 分钟）**：收到告警，确认影响范围，拉起应急群
2. **问题定位（10-30 分钟）**：查看 Sentry 日志，复现路径，检查最近发布
3. **临时方案（30 分钟-1 小时）**：功能降级、Feature Flag 回滚、CDN 回滚
4. **根本修复（1-2 小时）**：创建 hotfix 分支，修复，补充测试用例，双人 Code Review
5. **验证发布（30 分钟）**：预发布验证 → 灰度发布 → 监控观察
6. **复盘总结（24 小时内）**：撰写事故报告，改进措施，责任到人

## 10. H5 性能优化

| 方向 | 措施 |
|------|------|
| 网络 | 代码分割、懒加载、HTTP/2、CDN、Gzip/Brotli |
| 渲染 | 虚拟列表、transform 替代 top/left、requestAnimationFrame |
| 关键路径 | 内联关键 CSS、defer/async 加载 JS、Preload 预加载 |
| 图片 | WebP 格式、懒加载、响应式图片 |
| 缓存 | Service Worker、HTTP 强缓存 + 协商缓存 |
| JS | 防抖节流、Web Worker、事件委托 |
| 监控 | LCP < 2.5s、FID < 100ms、CLS < 0.1 |

**实际案例：**
- FCP：2.5s → 1.2s
- Bundle 体积：800KB → 200KB
- Lighthouse 评分：65 → 95
